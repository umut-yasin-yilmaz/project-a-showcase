# Before going public (showcase curation gate)

Use this list **before** [Step 0.5 — Flip showcase to public](distribution_plan.md#step-05--flip-showcase-to-public-when-vitrin-is-finalized) in `distribution_plan.md`.

## Content

- [ ] **README (EN):** intro, milestone, screenshots table, boundaries, downloads policy read cleanly—no “TODO” or draft phrases you do not want seen.
- [ ] **README (TR):** same story as EN; no contradictions.
- [ ] **`assets/showcase/`:** screenshots match the build you want to advertise (e.g. MVP `mvp-0.1`); filenames wired in README.
- [ ] **`docs/blueprint.md`**, **`qa_checklist.md`**, **`risk_register.md`:** safe for outsiders; no internal secrets or keys.
- [ ] **`distribution_plan.md`:** steps still match your intent (Steam handoff, Releases).

## Technical / GitHub

- [ ] Default branch is **`main`** and it is what you expect visitors to see first.
- [ ] No large binaries committed on `main` (use **Releases** for zips later).
- [ ] Latest changes **pushed** to GitHub before flipping visibility.

## After you click “Public”

- [ ] Open `https://github.com/umut-yasin-yilmaz/project-a-showcase` in **incognito / logged-out** window: README renders, images load.
- [ ] **Pin** the repo on your profile + optional **bio** / **website** ([Step 1](distribution_plan.md#step-1--github-profile-after-showcase-is-public)).

---

When every box you care about is checked → run **Step 0.5** (public visibility).
